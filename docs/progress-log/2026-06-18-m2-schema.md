# 2026-06-18 M2 schema de dominio implementado

## Contexto

Se implemento el slice `m2-domain-schema` en la branch `codex/m2-domain-schema`.
Este change fija el contrato de persistencia (modelos SQLAlchemy + migracion
Alembic) que habilita los slices siguientes de M2: repositories, ingestion,
retrieval y chat.

## Entregado

- Modelos SQLAlchemy para `projects`, `sources`, `documents`,
  `document_versions`, `chunks` y `chunk_sparse_embeddings` bajo
  `src/adaptive_rag/db/models/`.
- Migracion Alembic `6de20037eb1f_m2_domain_schema` con extension vector,
  tablas, foreign keys, check constraints e indices de aislamiento/filtering.
- Embedding denso `vector(1024)` sin HNSW (dense exacto como baseline de
  correctness, decision D2 del design).
- Tests unitarios de contrato de schema sobre SQLite.
- Tests de integracion contra Postgres + pgvector via testcontainers que
  validan la migracion, la dimension del vector y las columnas indexadas.

## Decisiones de implementacion

- `JSONWithJSONB`: TypeDecorator que usa JSONB en Postgres y JSON generico
  en SQLite, para que los tests unitarios puedan correr sin Postgres.
- `chunks.embedding`: `Vector(1024).with_variant(BLOB(), "sqlite")` para
  que la tabla se pueda crear en SQLite en los tests unitarios; la
  dimension exacta se valida en integracion.
- Integridad referencial de FKs se valida por contrato (NOT NULL + FK) en
  unit y por migracion real en integracion, ya que SQLite no enforcea FKs
  por defecto.
